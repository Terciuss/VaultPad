# VaultPad (Сейф данных)

[Репозиторий сервера](https://github.com/Terciuss/VaultPadServer)

Зашифрованное хранилище данных с архитектурой нулевого знания. Десктопное приложение для хранения паролей, ключей, документации и другой конфиденциальной информации. Все шифрование и дешифрование происходит локально — сервер никогда не видит открытые данные.

## Установка

Скачайте последний релиз из [GitHub Releases](https://github.com/Terciuss/VaultPad/releases).

### macOS

1. Скачайте `VaultPad-macOS-arm64.dmg` (Apple Silicon) или `VaultPad-macOS-x64.dmg` (Intel)
2. Откройте DMG и перетащите **VaultPad** в `/Applications`
3. Перед первым запуском откройте Terminal и выполните:
   ```bash
   xattr -cr /Applications/VaultPad.app
   ```
   Это необходимо, так как приложение не подписано Apple Developer ID. Без этого шага macOS заблокирует запуск.

   Альтернативно: **Системные настройки → Конфиденциальность и безопасность** → найти «VaultPad заблокирован» → нажать **Всё равно открыть**.

### Windows

1. Скачайте `VaultPad-Windows-x64-setup.exe`
2. Запустите установщик и следуйте инструкциям

### Linux

1. Скачайте `VaultPad-Linux-x64.AppImage`
2. Сделайте файл исполняемым и запустите:
   ```bash
   chmod +x VaultPad-Linux-x64.AppImage
   ./VaultPad-Linux-x64.AppImage
   ```

## Возможности

### Шифрование и безопасность

- **Шифрование с нулевым знанием** — AES-256-GCM с ключом на основе Argon2id, вся криптография выполняется в Rust
- **Мастер-пароль** — шифрует все файлы по умолчанию; никогда не покидает устройство
- **Смена мастер-пароля** — перешифровывает все V2-данные (файлы, бэкапы, verification token) новым ключом
- **Индивидуальные пароли файлов** — любой файл может использовать собственный пароль вместо мастер-пароля
- **Реестр паролей** — автоматически создаваемый зашифрованный мета-файл с кастомными паролями; синхронизируется между устройствами, позволяя расшифровывать файлы с отдельным паролем всем пользователям с мастер-паролем
- **Динамическое определение пароля** — тип шифрования определяется по формату данных (V2 = мастер-ключ, V1 = пользовательский пароль), без статических флагов
- **Интеграция с системным хранилищем** — мастер-пароль и пароли файлов хранятся в связке ключей ОС (macOS Keychain, Windows Credential Manager, Linux Secret Service)
- **Быстрая разблокировка PIN-кодом** — 4-значный PIN для быстрого доступа после первоначальной настройки; опционально
- **Автоблокировка** — настраиваемый таймер бездействия очищает расшифрованные данные из памяти

### Редактор

- **Rich-text редактор** — на основе TipTap с тулбаром: жирный, курсив, подчёркивание, зачёркивание, код, блоки кода, заголовки, выравнивание текста, цвет текста, выделение, очистка форматирования
- **Поиск и замена** — поиск текста с учётом регистра, замена одного или всех вхождений
- **Автосохранение** — отложенное сохранение контента с гарантированной записью при закрытии приложения
- **Перенос строк** — настраиваемый перенос длинных строк

### Хранилище и синхронизация

- **Работа без сети** — все данные хранятся в локальной базе SQLite; сервер не требуется
- **Многосерверная архитектура** — подключение к нескольким Go-серверам; для каждого сервера создаётся своя локальная база данных
- **Умная синхронизация** — отложенная отправка (60 с после редактирования), периодическая проверка (120 с), автоматическое определение online/offline
- **Разрешение конфликтов** — визуальный diff локальной и серверной версий, редактируемый предпросмотр, выбор версии или ручное слияние
- **Автоматические бэкапы** — создаются при значительных изменениях контента и перед синхронизацией; история версий с визуальным diff и восстановлением в один клик

### Администрирование

- **Админ-панель** — управление пользователями (создание, редактирование, удаление) на сервере
- **Шаринг доступа к файлам** — администраторы могут выдавать/отзывать доступ к отдельным файлам для пользователей
- **Автоматический сидер** — первый пользователь сервера создаётся автоматически

### Интерфейс

- **Локализация** — английский и русский через react-i18next
- **Темы оформления** — системная, светлая и тёмная
- **Нативное меню** — Настройки, Новый файл, Блокировка, Добавить сервер через системное меню ОС
- **Изменяемая ширина боковой панели** — перетаскиваемый разделитель с сохранением ширины

## Стек технологий

| Слой | Технология |
|---|---|
| Десктоп-фреймворк | [Tauri 2](https://v2.tauri.app/) (Rust + системный WebView) |
| Фронтенд | React 19, TypeScript, Vite 7, TailwindCSS 4 |
| Rich-text редактор | TipTap (StarterKit, Underline, Highlight, TextAlign, Color, TextStyle) |
| Управление состоянием | Zustand |
| Локализация | i18next + react-i18next |
| Шифрование | AES-256-GCM (`aes-gcm`), Argon2id (`argon2`) |
| Локальная БД | SQLite (`rusqlite`, встроенная) |
| Безопасное хранилище | Связка ключей ОС (`keyring`) |
| Безопасность памяти | `zeroize` для очистки ключей и паролей |
| Сервер | Go, MySQL, JWT-аутентификация |

## Требования

- **Node.js** 18+
- **Rust** 1.70+ (с `cargo`)
- **Системные зависимости** для Tauri 2: см. [требования Tauri](https://v2.tauri.app/start/prerequisites/)

## Начало работы

```bash
# Установка зависимостей фронтенда
npm install

# Запуск в режиме разработки (горячая перезагрузка фронтенда + Rust-бэкенд)
npm run tauri dev

# Сборка продакшн-версии
npm run tauri build
```

Собранное приложение будет в `src-tauri/target/release/bundle/`.

## Архитектура шифрования

```
Пользователь вводит мастер-пароль
        │
        ▼
Argon2id(пароль, соль) ──► 256-битный ключ
        │
        ▼
AES-256-GCM(ключ, nonce, открытый текст) ──► шифротекст
        │
        ▼
Хранится как: байт_версии || nonce(12) || шифротекст
```

- **Формат V2** (байт `0x02`): зашифровано кэшированным мастер-ключом (быстро, без Argon2id на каждую операцию)
- **Формат V1** (любой другой первый байт): зашифровано пользовательским паролем через полный вывод ключа Argon2id
- Для каждой операции шифрования генерируется новый случайный nonce
- Имена файлов тоже зашифрованы — сервер и база данных никогда не видят открытые имена

## Модель безопасности

- Мастер-пароль и производный ключ кэшируются в Rust `AppState` (за `Mutex`) во время разблокированной сессии
- Крейт `zeroize` очищает чувствительную память при блокировке и закрытии
- PIN валидируется через хэш Argon2id в связке ключей; PIN не используется для шифрования
- Связка ключей ОС защищает сохранённые пароли (macOS Keychain Access, Windows DPAPI, Linux Secret Service)
- Автоблокировка очищает все расшифрованные данные после настраиваемого периода бездействия
- Восстановление пароля отсутствует намеренно — потеря мастер-пароля означает потерю данных
- Смена мастер-пароля атомарно перешифровывает все V2-проекты, их бэкапы и verification token
- Реестр паролей — V2-зашифрованный мета-файл с кастомными паролями для совместного доступа; синхронизируется автоматически с приоритетом удалённых записей при конфликте

## Сценарий работы приложения

```
Запуск
  ├─ Есть сессия? ──► Есть PIN? ──► Разблокировка PIN ──► Главный экран
  │                       │
  │                       └─ Нет PIN ──► Авторазблокировка ──► Главный экран
  │
  └─ Нет сессии ──► Выбор БД ──► Установка мастер-пароля ──► Настройка PIN (опц.) ──► Главный экран

Серверный сценарий (из главного экрана):
  Добавить сервер ──► Авторизация (JWT) ──► Мастер-пароль сервера ──► Синхронизация
                                                                        │
                                                            ┌───────────┴───────────┐
                                                            │                       │
                                                      Нет конфликтов         Конфликты обнаружены
                                                            │                       │
                                                            ▼                       ▼
                                                      Файлы обновлены    Диалог разрешения конфликтов
                                                                        (diff, редактирование, выбор версии)
```
